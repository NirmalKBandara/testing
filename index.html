⏱️ Live Competition Protocol

1. Triage Phase ▼
Read all challenges first. Identify "quick wins" to get points on the board early. Don't dive deep until you know the landscape.
2. Confidence Building ▼
Start with categories or challenges you know best. Early solves build momentum and reduce panic.
3. Rabbit Hole Management ▼
Set strict time limits. If you are stuck for too long, move on. Don't let one challenge sink your ship.
4. Documentation ▼
Document everything you try, even failures. It prevents repeating mistakes and helps teammates.

🤝 Unit Cohesion (Teamwork)

A

Specialize

Divide and conquer. Each member should focus on specific categories (Web, Crypto, Pwn) to maximize efficiency.

B

Communicate

Share findings immediately. Avoid duplication of effort. If you find a hint for a different category, pass it on.

C

Shared Intelligence

Use tools like Notion, HackMD, or Google Docs to keep a live, shared notebook of all progress.

General Field Tips

01. READ SOURCE

Check challenge descriptions, filenames, comments, hidden fields, and referenced JS files.

02. TRY OBVIOUS

Always check default credentials (admin/admin), common passwords, and standard paths.

03. GOOGLE IT

Search error messages, weird strings, and specific headers. The answer is often indexed.
